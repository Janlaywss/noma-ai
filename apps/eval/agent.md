# Eval Agent 指南

本文件约束 `apps/eval` 内的后续修改。

## 测试原则

- 评估必须可重复，不依赖真实外部账号和实时网络状态。
- 使用固定时钟、固定 fixture 和可控 mock。
- 新增连接器或事件决策策略时，必须补充对应 eval。
- 失败用例要保留足够上下文，便于定位是连接器标准化、任务上下文还是决策策略问题。
- 当前 `src/index.ts` 是 smoke eval，后续复杂评估应新增 fixture 和独立断言模块，不要把所有逻辑堆在入口文件。

## 数据规则

- 不提交真实用户数据。
- 不提交 token、cookie、OAuth code、refresh token。
- 私有仓库、邮件、聊天消息必须脱敏到不可还原。
- fixture 中的时间、用户、仓库、邮箱使用稳定假数据。

## 断言规则

- 事件 Agent 输出必须按 schema 断言。
- 对副作用动作只断言动作意图和权限门禁，不调用真实 provider。
- 对幂等行为要验证重复事件不会重复执行。
- 对错误恢复要验证任务状态和事件日志摘要。
