01 Design principles

Five rules we lean on whenever a screen gets ambiguous.

①

Make the agent legible

Every action Noma takes shows up as a card in the conversation, an entry in the inbox, or a row in the audit log. No invisible plumbing. If the user can't see what Noma did and why, we've failed.

②

One conversation per intent

A session pairs a chat with the connectors + tasks it owns. Switching sessions = switching context. Like Messages or Linear cycles, not like a Slack channel firehose.

③

Structured events, structured tags

Everywhere events surface — chat, inbox, decision chain — the same vocabulary repeats: EVENT · CONTEXT · DECIDE · ACT. Predictable tags = predictable mental model.

④

Local-first, calm-first

Data lives on disk. The UI doesn't shout — it uses small live-dots and pills, not red banners. Reserve red for "needs you now."

⑤

Sketch first, decide later

This system intentionally reads as low-fi. The hand-written annotations and 2px ink borders are a permission slip to keep iterating. Don't promote anything to "final" until the flow is locked.

If a screen looks polished, users start critiquing the gradient instead of the flow. Stay sketchy ✏
The sketchy aesthetic is a strategic choice for this phase. We will swap to clean shadcn surfaces (no ink shadows, no Caveat) once the structure ships.

02 Voice & tone

How Noma talks, in chat, in cards, in error states.

Noma's voice

  • Plain spoken. "Got it. Setting up a task that listens to two connectors." — not "Affirmative. Initializing event subscription pipeline."
  • First person, present tense. "I sent a Lark message" — not "Noma has dispatched a message."
  • Concrete > abstract. Quote the value: "NVDA dropped to $99.84", not "price threshold breached".
  • One verb per card. Action cards lead with: Sent, Created, Skipped, Watching, Asked.

Microcopy patterns

empty state Nothing here yet — the first event will land in this view. Friendly + tells you what to expect
awaiting input I drafted a reply. Send it as-is, edit, or discard? Always give 3 verbs, not yes/no
filtered 3 events processed · 1 forwarded to you Receipts for what got skipped
error Stocks connector timed out — retrying in 30s. State the cause + next step

Color rules

  • Accent blue is for human annotation only. Never use it for primary buttons, links, or product chrome — that role belongs to ink (near-black). Blue marker = "designer is talking to you."
  • Yellow is reserved for sticky notes. Don't use it for warnings — that's amber/orange in the pill scale.
  • Status colors are pills, not surfaces. A green page is wrong. A green pill that says "live" is right.
  • Connector brand colors live inside the connector icon only. They never bleed into the surrounding card.

08 Layout patterns

The frame structures that repeat across screens.

App shell

Every screen sits inside a MacFrame: 32px traffic-light titlebar, then a body that's a flex row of .app-sidebar + .app-content.

  • Sidebar — 220–260px, soft-grey bg (oklch(.98 0 0)), 1px line border-right.
  • Content — fills, scrolls. First child is a 48px .app-header with title + meta + actions.
  • Optional right rail — 280px, line border-left. Used sparingly; default is no rail (chat A pattern).

App-header recipe

NVDA earnings watch 2 connectors · last event 3m ago
● live

List + reading pane (inbox / mail style)

Three columns: sidebar · list (320–380px) · reading pane (fills). Selected row = soft-grey bg, ink border-left.

Kanban (tasks)

Three vertical columns: Running, Waiting on you, Done today. Each column has a header pill with count. Cards are 1px soft-line, 8px radius, padding 12px.

Decision chain (task detail)

Vertical timeline. Each event is a node block: tag pill (EVENT / CONTEXT / DECIDE / ACT) + title + content card + connector chip rail at the bottom. Use the same 4 tags everywhere — never invent a 5th.

Window scaling

Artboards are fixed 1280×820 (16:10ish). On the canvas they're scaled to fit; in focus mode they fill the viewport letterboxed on warm-paper background.

11 Wireframe overlay

The "design-in-progress" treatment that sits on top of the real interface tokens.

What makes it look sketchy

  1. Warm paper background — oklch(0.97 0.005 80). Not pure white.
  2. Ink borders — 1.5px solid --ink on hero elements (windows, sketch cards). 1px --line for everything else.
  3. Solid offset shadows — 1.5–3px ink shadow with no blur. Reads as paper-on-paper.
  4. Hand-written annotations — Caveat in --accent (blue marker), tied to the design with curved SVG arrows.
  5. Sticky-note callouts — yellow paper, 1° rotation, soft drop shadow.
  6. Diagonal-hatch placeholders — for any image/avatar we haven't built yet. Always paired with a Caveat label.
  7. "Drop a .js file"-style affordances — dashed boxes for things that are intentionally empty.

Annotation arrows

4 directions: up, down, left, right. Drawn as SVG curves with two small bezier control points so they look hand-drawn, never as straight lines or right angles. Stroke 1.5px, blue marker color.

Promotion path (lo-fi → hi-fi)

When a wireframe is approved and we're ready to move to a real screen:

  1. Strip the warm paper bg → use shadcn --background.
  2. Strip ink borders + offset shadows → use --border + --shadow-sm.
  3. Replace Caveat annotations with proper UI labels or remove.
  4. Remove sticky notes (capture rationale in PR description).
  5. Replace placeholder glyphs with Lucide icons.
  6. Keep the structure — sidebar widths, header height, action card pattern, status vocabulary.

12 Do / Don't

Common pitfalls when iterating on Noma screens.

Do
  • Use ink (near-black) for primary buttons.
  • Lead action cards with one verb: Sent, Created, Skipped.
  • Show what got skipped, not just what got handled.
  • Use the EVENT / CONTEXT / DECIDE / ACT tag set.
  • Quote concrete values in chat ("$99.84"), not abstractions.
  • Reserve red exclusively for "needs you now."
  • Make connectors clickable everywhere they appear.
Don't
  • Don't use the blue accent for product UI — that's annotation only.
  • Don't add a 5th status type without updating this guide first.
  • Don't hide what the agent did inside a collapsed log.
  • Don't use yellow as a warning color (yellow = sticky note).
  • Don't introduce a 4th typeface.
  • Don't put gradients on surfaces. Flat is the goal.
  • Don't "polish" wireframes — that buys premature commitment.
