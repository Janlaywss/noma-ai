Noma — 设计风格指南

01 设计原则

五条原则，每当某个屏幕陷入分歧时回来对照。

①

让 Agent 看得见

Noma 做的每一件事都会以卡片出现在对话里、条目出现在 inbox 里、行出现在审计日志里。没有看不见的管道。如果用户看不到 Noma 做了什么、为什么这样做，我们就失败了。

②

一个意图一个会话

一个 session 把一段对话和它所拥有的连接器+任务绑在一起。切换 session = 切换上下文。像 iMessage、Linear 的循环，不像 Slack 那种信息洪流。

③

结构化事件，结构化标签

事件出现的所有地方——chat、inbox、决策链——都用同一套词汇：EVENT · CONTEXT · DECIDE · ACT。可预期的标签 = 可预期的心智模型。

④

本地优先，平静优先

数据存在本地。UI 不喧哗——用小绿点和 pill，不用红色横幅。红色保留给"现在就需要你"。

⑤

先草图，再决定

这套系统刻意保持低保真。手写注释、2px 墨水边框是一张"继续迭代"的许可证。在流程定下来之前，不要把任何东西"提升"成最终稿。

屏幕一旦看起来精致，用户就开始评论渐变色，而不是流程本身。保持草图感 ✏
草图美学是这个阶段的战略选择。等结构定下来，我们会切到干净的 shadcn 表面（去掉墨水阴影、Caveat 字体）。

02 语气与表达

Noma 在 chat、卡片、错误状态里如何说话。

Noma 的语气

  • 口语化。"知道了，我建一个监听两个连接器的任务"——而不是"已确认。正在初始化事件订阅管线"。
  • 第一人称、现在时。"我已经发了一条 Lark 消息"——而不是"Noma 已派发消息"。
  • 具体 > 抽象。引用真实数值："NVDA 跌到 $99.84"，而不是"价格阈值已突破"。
  • 每张卡片只用一个动词。action card 开头：已发送、已创建、已跳过、正在监听、正在询问。

微文案模式

空状态 还没有内容 — 第一个事件会出现在这里。 友好 + 告知接下来会发生什么
等待你确认 我起草了回复。直接发送、修改、还是丢弃？ 永远给三个动词，不是 yes/no
已过滤 处理了 3 个事件 · 1 个转发给你 为被跳过的事件留收据
错误 Stocks 连接器超时 — 30 秒后重试。 说明原因 + 下一步

用色规则

  • 蓝色点睛色只用于人为注释。永远不要用作主按钮、链接、产品 chrome——那些角色属于墨色（近黑）。蓝记号笔 = "设计师在跟你说话"。
  • 黄色保留给便签。不要用作警告色——警告用 pill 阶梯里的橙色。
  • 状态色是 pill，不是面色。整页绿色是错的。一颗写着 "live" 的绿色 pill 才是对的。
  • 连接器品牌色只活在连接器图标内部。绝不溢出到周围的卡片上。

04 字体

密度模式

两种密度，由 body 上的 data-density 控制。Comfy = 默认，Compact = 高级用户。

  • Comfy（舒适）：侧边栏项 32px 高 · 行间距 8px · 卡片 padding 16px
  • Compact（紧凑）：侧边栏项 26px 高 · 行间距 4px · 卡片 padding 12px

08 布局模式

在屏幕之间反复出现的框架结构。

App Shell

每个屏幕都装在 MacFrame 里：32px 红绿灯标题栏，下面是一个由 .app-sidebar + .app-content 组成的横向 flex。

  • 侧边栏 — 220–260px，soft 灰底（oklch(.98 0 0)），1px line 右边框。
  • 内容区 — 自适应、可滚动。第一个子元素是 48px 的 .app-header，包含标题 + meta + 操作。
  • 可选右栏 — 280px，1px line 左边框。慎用；默认无右栏（chat A 模式）。

App-header 配方

NVDA 财报监控 2 个连接器 · 上次事件 3 分钟前
● live

列表 + 阅读窗（邮件式）

三栏：侧边栏 · 列表（320–380px）· 阅读窗（自适应）。选中行 = soft 灰底 + 墨色左边框。

看板（任务）

三个纵向列：运行中、等待你确认、今日已完成。每列顶部有带计数的 pill。卡片 1px soft 边框，8px 圆角，padding 12px。

决策链（任务详情）

纵向时间线。每个事件是一个节点块：tag pill（EVENT / CONTEXT / DECIDE / ACT）+ 标题 + 内容卡片 + 底部连接器 chip 行。永远只用这 4 个 tag——不要发明第 5 个。

窗口缩放

Artboard 固定 1280×820（约 16:10）。在画布上自动缩放适配；focus 模式下铺满视口，warm-paper 背景做信箱式留边。

11 线框图叠层

叠在真实界面 token 之上的"设计中"处理。

是什么让它看起来像草图

  1. 暖纸色背景 — oklch(0.97 0.005 80)。不是纯白。
  2. 墨水边框 — 主要元素（窗口、sketch 卡片）用 1.5px solid --ink。其他用 1px --line。
  3. 实心偏移阴影 — 1.5–3px 墨水阴影，无模糊。读起来像纸压纸。
  4. 手写注释 — Caveat 字体 + --accent 蓝记号笔色，用 SVG 曲线箭头连到设计上。
  5. 便签提示 — 黄色纸张，旋转 1°，柔和投影。
  6. 斜线占位符 — 任何还没做的图片/头像。永远配一个 Caveat 标签。
  7. "拖入 .js 文件"式占位 — 故意留空的虚线框。

注释箭头

4 个方向：up、down、left、right。用 SVG 曲线绘制，带两个小贝塞尔控制点，让它看起来像手绘的，绝不是直线或直角。线宽 1.5px，蓝记号笔色。

升级路径（低保真 → 高保真）

当一张线框图通过审核、准备升级为正式屏幕时：

  1. 去掉暖纸色背景 → 改用 shadcn --background。
  2. 去掉墨水边框 + 偏移阴影 → 改用 --border + --shadow-sm。
  3. 把 Caveat 注释替换为正式 UI 标签或直接删掉。
  4. 移除便签（设计理由记录到 PR 描述里）。
  5. 把占位字形换成 Lucide 图标。
  6. 保留结构——侧边栏宽度、header 高度、action card 模式、状态语汇。

12 该做 / 不该做

迭代 Noma 屏幕时常见的陷阱。

该做
  • 主按钮用墨色（近黑）。
  • Action card 开头用一个动词：已发送、已创建、已跳过。
  • 把跳过的事件也展示出来，不只是处理过的。
  • 用 EVENT / CONTEXT / DECIDE / ACT 这套标签。
  • 聊天里引用具体数值（"$99.84"），不要抽象表述。
  • 红色严格保留给"现在就需要你"。
  • 连接器在出现的所有地方都可点击。
不该做
  • 不要把蓝色点睛色用在产品 UI 上——那只属于注释。
  • 没更新本指南之前，不要新增第 5 种状态。
  • 不要把 Agent 做的事藏到折叠日志里。
  • 不要用黄色作警告色（黄色 = 便签）。
  • 不要引入第 4 套字体。
  • 表面上不要加渐变。扁平是目标。
  • 不要"打磨"线框图——那会买来过早的承诺。
